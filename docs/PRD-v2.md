# Very Boring V2 — 产品文档

> "Si j'avais cinquante-trois minutes à dépenser, je marcherais tout doucement vers une fontaine..."
> — Le Petit Prince

---

## 1. 产品定位

Very Boring 是一个**抽卡叙事器**。

不是推荐引擎。不是待办清单。不是效率工具。

它解决的问题是：**当你终于忙完了，整个人瘫坐在沙发上，不知道该做什么，也不想刷手机——这时候，什么能进入你的心中？**

答案是一张牌。一张随机抽到的、带着神秘符号的牌。它会带你走一段很短的路，然后在终点，给你一张皱巴巴的小纸条，上面写着一个秘密——其实是一件好玩的、现在就可以做的事情。

> 在用户"不知道自己要什么"的时刻，通过仪式感 + 随机性 + 叙事游戏，创造一个从忙碌到在场的过渡。

---

## 2. 核心循环

```
打开应用 → 五张牌静静排列 → 选一张 → 文字缓缓出现 → 做一个选择 → 文字再出现 → 再选择 → 皱巴巴的小纸条翻出来 → 收起来 或 丢掉
```

整个体验 **2-3 分钟**。然后关掉标签页。

下一次瘫在沙发上时，你会想起——"也许我可以抽张牌。"

---

## 3. 产品原则

### 3.1 不提供列表

从五张牌中选一张。不是选择"最好的"，而是选择"被吸引的"。

### 3.2 仪式感 > 效率

每一步都有节奏：牌的出现、文字的流动、选择时的停顿。不急。

### 3.3 发现 > 推荐

小纸条上的内容不是"建议你去做 X"。而是"你家里某个角落，藏着一本你想读很久的书"。是一个秘密，不是一条指令。

### 3.4 足够好，不完美

抽到的牌是随机的。但随机中有深意——因为牌的元素（金木水火土）会给叙事一个隐性的方向。用户永远不知道下一次会得到什么。

### 3.5 极度有边界

这是有边界的冥想。不是开放式探索。3-5 步选择，然后结束。不会让你沉迷，只会让你微微心动。

## 4. 核心功能

### 4.1 牌面系统

#### 4.1.1 符号库

一个巨大的抽象 SVG 符号库，每个符号有一个诗意的名字：

**金 (Metal)**
镜 · 锋 · 铃 · 锁 · 针 · 冠 · 链 · 币 · 碑 · 匕

**木 (Wood)**
树 · 森 · 枝 · 根 · 藤 · 叶 · 芽 · 茧 · 棚 · 种

**水 (Water)**
泉 · 雨 · 潮 · 雾 · 井 · 冰 · 渊 · 露 · 溪 · 泪

**火 (Fire)**
焰 · 耀 · 炬 · 灰 · 烛 · 炬 · 耀斑 · 烬 · 星 · 炬

**土 (Earth)**
尘 · 岩 · 沙 · 壤 · 谷 · 壁 · 痕 · 径 · 阶 · 冢

每个符号：

- 有一个唯一的抽象 SVG 图案（几何、极简、带有一点神秘感）
- 属于金木水火土之一
- 作为叙事生成的"种子"（seed）

#### 4.1.2 元素系统

五元素（金木水火土）是一个**隐性分类系统**：

- **视觉表现**：不同元素的牌有微妙的边框色调差异
  - 金：冷银灰边框
  - 木：深绿边框
  - 水：深蓝边框
  - 火：暖橙红边框
  - 土：赭石棕边框
- **LLM 影响**：元素会引导叙事的基调
  - 金 → 精确、结构、秩序
  - 木 → 成长、自然、生机
  - 水 → 流动、情感、直觉
  - 火 → 能量、行动、热情
  - 土 → 沉稳、落地、踏实
- **用户不需要知道这个系统的存在**。他们只会感觉到"这张牌给我的感觉不太一样"。

#### 4.1.3 抽牌机制

- 每次打开应用，从符号库中随机抽取 **5 张牌**
- 保证 5 张牌中元素尽量不重复（最多 2 张同元素）
- 牌面静静排列在屏幕中央，有微妙的浮动动画
- 用户点击选择一张

### 4.2 叙事游戏

选牌后进入叙事游戏。格式为：

```
[一段氛围文字 2-3 句] → [两个选项] → 用户选择 → [回应 + 新场景 2-3 句] → [两个选项] → ... → 小纸条
```

#### 4.2.1 叙事规格

- **步数**：3-5 步选择
- **文字长度**：每段 2-3 句话，简洁、有氛围
- **选项**：每步 2 个选项，都有意义、都有效、都不"错"
- **总时长**：90 秒 - 2 分钟
- **语言**：中文

#### 4.2.2 叙事输入

生成叙事时，系统会收集以下上下文作为 LLM 输入：

| 输入            | 来源                   | 用途             |
| --------------- | ---------------------- | ---------------- |
| 牌面符号 + 元素 | 随机抽取               | 叙事主题种子     |
| 当前时间        | `new Date()`           | "夜深了"、"午后" |
| 星期几          | `new Date()`           | "周一"、"周末"   |
| 季节            | 从日期推断             | "深秋"、"初春"   |
| 天气            | OpenWeatherMap 免费层  | "窗外下着雨"     |
| 地理位置        | 城市名（用户可选配置） | 本地化氛围       |

天气和地理为可选功能。MVP 中，时间 + 星期 + 季节 足够。

#### 4.2.3 生成策略

**Hybrid 方案**：

1. 用户选牌后，立即请求生成**第一步场景 + 两个选项**（非流式，快速返回，减少等待）
2. 用户做出选择后，**流式生成**下一步场景 + 选项（SSE，文字逐字出现，有"活着"的感觉）
3. 重复直到最后一步
4. 最后一步生成**小纸条**（流式，有仪式感地逐字显现）

### 4.3 小纸条

叙事的终点是一张**皱巴巴的小纸条**。

#### 4.3.1 内容格式

- 第二人称、现在时："你家中的某个位置..."
- 1-3 句话
- 具体、可执行，但不是待办清单
- 语气像一个朋友低声告诉你一个秘密

**示例**：

> "我想，你家中的某个位置，藏着一本你想读很久但一直没有开始的书。你会感兴趣那个故事吗？"

> "你附近的某条街，有一家很小的店，门口没有招牌。如果你现在走过去，也许会遇到什么。"

> "你上次站在窗前看了多久？今天的天空值得再看一次。"

> "你的手机里有一首歌，你已经很久没听了。但它一直在等你。"

#### 4.3.2 视觉表现

- 纸条从牌的下方缓缓翻出（动画）
- 微微倾斜（2-3 度），有纸张质感
- 字体带有手写感（或使用衬线体营造"旧"的感觉）
- 背景为泛黄/米色的纸张色

#### 4.3.3 操作

- **收起来** → 保存到"纸条盒"（本地存储）
- **丢掉** → 纸条消失（有动画：飘走或碎裂）
- **再来一张** → 回到首页，重新抽牌

### 4.4 纸条盒（收藏）

用户的"鞋盒"——存放收起来的纸条。

- 简单的列表/网格展示
- 每张纸条显示：符号名、日期、内容
- 没有计数、没有进度、没有成就系统
- 就是一个安静的盒子

## 5. 页面结构

V2 只有两个页面：

### 5.1 主页面 `/`

核心体验的唯一入口。没有分类、没有导航。只有：

- 五张牌（或正在进行的叙事）
- 极简的页脚（设置入口）

### 5.2 设置页面 `/settings`

- LLM 供应商配置（沿用 V1 的 9 供应商支持）
- 城市名配置（可选，用于天气）
- API Key + 模型选择 + 连接测试

设置入口极度克制——页脚一个小小的齿轮图标。

## 6. 数据模型

### 6.1 符号（Symbol）

```typescript
interface Symbol {
  id: string; // 唯一标识，如 'mirror'
  name: string; // 中文名，如 '镜'
  element: Element; // 所属元素
  svgPath: string; // SVG 图案路径
}
```

### 6.5 元素（Element）

```typescript
type Element = 'metal' | 'wood' | 'water' | 'fire' | 'earth';

interface ElementConfig {
  id: Element;
  name: string; // 金、木、水、火、土
  borderColor: string; // 牌面边框色调
  tone: string; // LLM 提示词中的基调描述
}
```

### 6.3 牌面（Card）

```typescript
interface DrawnCard {
  symbol: Symbol;
  position: number; // 1-5 的位置
}
```

### 6.4 叙事状态（NarrativeState）

```typescript
type NarrativePhase = 'drawing' | 'intro' | 'playing' | 'note' | 'done';

interface NarrativeStep {
  sceneText: string;
  options: Array<{ id: string; text: string }>;
  choice?: string; // 用户的选择
}

interface NarrativeState {
  phase: NarrativePhase;
  drawnCards: DrawnCard[];
  selectedCard?: DrawnCard;
  steps: NarrativeStep[];
  note?: string; // 最终的小纸条内容
}
```

### 6.5 小纸条（Note）

```typescript
interface SavedNote {
  id: string;
  symbolName: string; // 来自哪张牌
  element: Element;
  content: string; // 纸条内容
  createdAt: string; // ISO 日期
}
```

## 7. LLM Prompt 策略

### 7.1 叙事生成 Prompt

系统 prompt 包含：

- 牌面符号名 + 元素 + 元素基调
- 当前上下文（时间、天气、季节）
- 叙事规则（步数、格式、选项数量、语气）
- 输出格式要求（JSON：scene_text, options）

### 7.2 小纸条生成 Prompt

基于完整的叙事历史（用户在每个步骤的选择）+ 牌面 + 上下文，生成最终纸条。

### 7.3 Prompt 版本管理

沿用 V1 策略：prompts 放在 `src/lib/prompts/` 下，按版本命名。

## 8. API 设计

### 8.1 `POST /api/narrative/start`

选牌后调用。返回第一步场景 + 选项。

### 8.2 `POST /api/narrative/step`

每步选择后调用。接收选择 + 叙事历史，流式返回下一步。

### 8.3 `POST /api/narrative/conclude`

最后一步。流式返回小纸条。

### 8.4 `GET /api/weather?city=xxx`

代理天气查询（OpenWeatherMap 免费层）。可选。

### 8.5 `POST /api/test-connection`

沿用 V1，测试 LLM 连接。

## 10. 与 V1 的关系

V2 是**全新产品**，不是 V1 的迭代。核心变化：

|          | V1                      | V2               |
| -------- | ----------------------- | ---------------- |
| 入口     | 4 个分类（吃/动/做/去） | 单一：抽牌       |
| 输入     | 回答问题                | 选牌 + 叙事选择  |
| 输出     | 生活片段（微剧本）      | 小纸条（秘密）   |
| 感觉     | 工具（帮你决定）        | 仪式（帮你在场） |
| 时长     | 30 秒                   | 2-3 分钟         |
| 回访理由 | 需要做决定时            | 想抽张牌时       |

**V1 代码保留**（git history），V2 在同一仓库中重建。可复用的部分：

- LLM provider 系统（9 供应商 + 双协议）
- 设置页面基础框架
- 连接测试 API
- localStorage 存储工具
- 设计 token / 动画基础设施
