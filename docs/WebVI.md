Web Visual Identity (Web VI)

Version 1.0
Design Philosophy: Calm Functionalism

⸻

1. Design Philosophy

1.1 Core Principle

Design is the control of attention, not decoration.

The system prioritizes:
	•	High aesthetic density
	•	Low cognitive load
	•	Strong intentional flow

⸻

1.2 Experience Model

Single-threaded Interaction

At any given moment, the user should only:
	•	Focus on one goal
	•	Process one primary piece of information
	•	Take one primary action

⸻

1.3 Three Foundational Rules

(1) Clarity over decoration

Visual elements must communicate structure, not style.

(2) Focus over abundance

Remove anything that competes for attention.

(3) Timing over richness

Interactions should appear only when needed.

⸻

2. Page Typology

All pages must be classified into one of two types.

⸻

2.1 Narrative Pages (Story-driven)

Used for:
	•	Landing pages
	•	Product introductions
	•	Brand storytelling

Characteristics:
	•	Full-screen sections
	•	Scroll-driven progression
	•	Minimal navigation
	•	Strong visual hierarchy

Rule:

One section = one idea

⸻

2.2 Functional Pages (Task-driven)

Used for:
	•	Dashboards
	•	Documentation
	•	Settings / workflows

Characteristics:
	•	Dense but structured information
	•	Clear navigation
	•	Fast scannability
	•	Grid-based layout

Rule:

Optimize for speed and clarity, not immersion

⸻

3. Typography System

3.1 Font Strategy
	•	Maximum 2 font families
	•	Serif (e.g. Georgia) for emphasis / headings
	•	Sans-serif for body and UI

⸻

3.2 Hierarchy

Level	Size	Usage
H1	32–48px	Hero / page title
H2	24–32px	Section titles
H3	18–22px	Subsections
Body	14–16px	Main content
Caption	12–13px	Secondary info


⸻

3.3 Rules
	•	Line height: 1.4–1.6
	•	Use weight, not color, for hierarchy
	•	Avoid mixing too many font sizes
	•	Ensure Chinese-English compatibility

⸻

4. Spacing System

4.1 Grid System
	•	Base unit: 8px
	•	All spacing must follow multiples of 8

⸻

4.2 Spacing Scale

Type	Size
Section spacing	64–120px
Block spacing	24–40px
Element spacing	8–16px


⸻

4.3 Principle

Spacing defines structure, not emptiness.

No arbitrary spacing is allowed.

⸻

5. Color System

5.1 Structure
	•	1 Primary color
	•	1–2 Secondary colors
	•	Neutral grayscale system (dominant)

⸻

5.2 Usage Rules

Color is only used for:
	•	Primary actions (CTA)
	•	Status (success / error / warning)
	•	Key emphasis

⸻

5.3 Restrictions
	•	No decorative gradients unless meaningful
	•	Avoid multi-color interfaces
	•	Backgrounds should remain neutral

⸻

6. Layout System

6.1 Structural Principle

Layout is a narrative flow, not a container of elements.

⸻

6.2 Layout Patterns

Narrative Layout
	•	Full-width sections
	•	Centered content
	•	Strong vertical rhythm

Functional Layout
	•	Grid-based (12-column recommended)
	•	Left navigation or top navigation
	•	Modular blocks

⸻

6.3 Constraints
	•	Avoid visual clutter
	•	Avoid competing focal points
	•	Every screen must have a clear visual entry point

⸻

7. Interaction Design

7.1 General Principle

Interaction should support understanding, not attract attention.

⸻

7.2 Motion Guidelines
	•	Duration: 150–300ms
	•	Easing: smooth, non-bouncy
	•	Avoid simultaneous animations

⸻

7.3 Interaction Types

(1) Functional Feedback (Required)
	•	Hover states
	•	Click states
	•	Loading indicators

⸻

(2) Contextual Interaction (Controlled)
	•	Modal (preferred for focus)
	•	Dropdown
	•	Inline expansion

⸻

(3) Expressive Interaction (Limited)
	•	Scroll animation
	•	Visual transitions

Only allowed in narrative pages.

⸻

8. Focus Management

8.1 Core Rule

The interface must actively guide user attention.

⸻

8.2 Implementation

1. One Primary Action per Screen
	•	Only one dominant CTA

⸻

2. Visual Hierarchy
	•	Primary content: high contrast
	•	Secondary content: muted
	•	Background: minimal

⸻

3. Attention Control Methods

Method	Use Case
Fullscreen	Critical focus
Modal	Interrupt with purpose
Inline	Maintain flow
Tooltip	Minor info


⸻

8.3 Anti-patterns
	•	Floating random UI elements
	•	Competing CTAs
	•	Hidden important actions

⸻

9. Component Principles

9.1 Buttons
	•	Clear hierarchy: Primary / Secondary / Ghost
	•	No excessive styling
	•	Consistent padding and radius

⸻

9.2 Cards
	•	Used only when grouping is necessary
	•	Avoid overuse
	•	Prefer spacing over borders

⸻

9.3 Forms
	•	Minimal fields
	•	Clear progression
	•	Immediate validation feedback

⸻

10. Content Design

10.1 Writing Principles
	•	Short, direct, precise
	•	Avoid decorative language
	•	Each sentence must serve a purpose

⸻

10.2 Information Density
	•	Narrative pages: low density
	•	Functional pages: optimized density

⸻

11. Design Process

Step 1: Define Page Type

Narrative vs Functional

⸻

Step 2: Define User Intent

What is the single thing the user needs to do here?

⸻

Step 3: Remove Elements

For each element:
	•	Is it necessary?
	•	Does it distract?
	•	Can it appear later?

⸻

Step 4: Apply Visual System

Only after structure is finalized:
	•	Typography
	•	Spacing
	•	Color

⸻

12. Summary

This system is defined by:
	•	Minimalism without emptiness
	•	Structure without rigidity
	•	Interaction without noise

⸻

Final Definition

Calm Functionalism =
Structured clarity + Controlled attention + Intentional interaction
